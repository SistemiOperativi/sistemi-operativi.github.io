Requirements¶

To submit a solution it is required:

  1. Get a GitHub account

  2. Accept the welcoming assignment: https://classroom.github.com/a/qU7ZFhmv

  3. Associate the GitHub account to the id received via email

  1. Look for your id by using the search function on your browser (crtl+f has been tested on firefox and google chrome)

  2. Click on your id

  3. Check that it is the correct id

  4. Click Ok

  1. Accept the GitFundamentals assignment

Challenge 1: The Lecturer’s Password¶

State:OPEN

During the lecture 22/12/2021 - C examples, security aspects and introduction to Linux, the lecturer discussed several aspects about security. In particular, he introduced the /etc/passwd/ and /etc/shadow files in GNU/Linux systems. Moreover, he explained several details about permissions and contents of the shadow file. In fact, he described how to pinpoint:

  • the hash of a user password;

  • the salt;

  • the hash algorithm.

After 81 minutes and 30 seconds of the lecture, he showed the shadow file from his own device and the entry related to its user.

The challenge¶

Discover the lecturer’s password.

The prize¶

  • 1 point to the final exam for the first solution that gets the password

The rules¶

To sumbit a solution it is required:

  1. Accept the assignment: https://classroom.github.com/a/oRfof8ZI

  2. publish on the assignment repository maintaing the code used to discover the password

  3. to send an email to the lecturer with [OS2122-CHALLENGE-1] as object and containing:

  1. the hash of the commit which maintains the code to be submitted for review

  2. your id

A solution is valid if it meets the following requirements:

  1. it has been submitted correctly

  2. it does NOT exploits programs for password crack/recovery (e.g. Hashcat, John the Ripper). It is allowed to use the programs in the following to built your solution:

  • openssl

  • bash

  • GNU crypt

  • crypt

Finally, it is allowed to submit up to one solution. If multiple solution have been submitted, only the first one in chronological order will be considered.

Deadlines¶

  • The challenge remains open until the points have not been assigned.

  • The points must be gained within the A.A. 2021/2022.
