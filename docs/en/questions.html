Training exercises¶

This section contains several exercises/questions that allows you to test your knowledge and skills.

Questions¶

1 - Introduction¶

  1. Describe the concept of interrupt and show its usage in modern operating system.

  2. Describe the main data structures used by an operating systems.

  3. Describe the main beneficts and challenges of multiprogramming.

  4. What are the system calls and what relationship do they have with system libraries?

  5. Show examples of hardware supports required by the operating systems to provide the protection of system resources.

  6. What are the main purposes of an operating systems.

  7. Why is the distinction between kernel mode and user mode a form of protection?

  8. Describe 3 services offered by an operating system and explain why they are convenient for users. Also, why is it inconvenient that they are provided by user programs

  9. Descrivere le principali caratteristiche di un sistema batch e di un sistema batch multiprogrammato, discutendo anche in modo comparativo i vantaggi o svantaggi dell’uno verso l’altro.

2- Processes and threads¶

  1. Describe the concept of mode switch and process switch.

  2. Describe the concept of process. What are the possible states of a process?

  3. Show and explaing the main differences between processes and threads.

  4. When a process is created via fork, what is shared betweenparent and child process

  5. What is the difference between User-Level Thread and Kernel-Level Thread?

  6. When a thread is created via pthread_create, what is shared between parent and child threads?

3 - Scheduling¶

  1. What are the main evaluation criteria of a cpu scheduler?

  2. What are the main challenges in CPU-scheduling introduced by multicores.

  3. What is the difference between preemtive and non-preemptive scheduling.

  4. Describe the Shortest-Job First policy and discuss its pros and cons.

  5. Describe the Round Robin policy and its pros and cons.

  6. Describe the Fair-Share policy and the problems it solves.

  7. Describe the Linux scheduler.

  8. Describe the Multilevel Feedback Queue scheduler.

  9. What is the advantages of setting different time-slices in a Multilevel queues scheduling algorithm?

  10. How can we set up a priority scheduler to mimic the Shortest Job First and the First Come First Serve schedulers?

  11. Descrivere lo scheduler di CPU round-robin virtuale, discutendone vantaggi e svantaggi rispetto allo scheduler round-robin.

  12. Si descrivano gli scheduler di CPU Shortest-Process-Next (SPN) e la sua variante Shortest–Remaining-Time-Next (SRTN), evidenziandone i vantaggi e gli svantaggi.

4 - Synchronization¶

  1. What is a Read-Modify-Write instruction?

  2. Describe the lock synchronization primitive and the properties of correctness and progress it must guarantee.

  3. Describe the Bakery algorithm.

  4. Are spinlocks appropriate in a single-core system? Motivate your answer.

5 - Memory management¶

  1. Describe pagination and segmentation techniques, showing their pros and cons.

  2. Show a scheme of two-level paging and the process of translating a linear address to a physical address. Briefly discuss the main cons and pros of this approach.

  3. What is a translation lookaside buffer? What is its role?

  4. Show a scheme for paged segmentation with TLP and describe how it works.

  5. What is a buddy system?

  6. Describe the Belady’s anomaly.

  7. Describe the clock algorithm for page replacement.

  8. Describe the main aspects of virtual-memory management.

  9. Describe page-fault handling with virtual memory based on pagination.

  10. Describe the main aspects of policies for resident-set management.

  11. Describe the thrashing phenomena related to virtual memory.

  12. Describe the memory management technique based on dynamic partitions. Then, indicate which supports are required for address binding

  13. Describe the Least-Recently-Used (LRU) Selection Algorithm for for systems with pagination-based virtual memory.

  14. Describe the pagination technique and the required data structures that the operating system need to manage for it.

  15. Describe the optimal algorithm for page replacement for virtual memory. Does it suffer from the Belady’s anomaly? Motivate your answer.

  16. Describe the page table data structure, its usage and its relation with hardware components of conventional hardware.

6 - I/O Management¶

  1. What are the challenges introduced by Direct Memory Access with respect to process management?

  2. Describe the I/O buffering and the problems it solves.

  3. Show the main challenges to be faced while developing I/O-scheduling policies.

  4. Describe First-Come-First-Served and Shortest-Seek-Time-First policies for disk-scheduling. Discuss their main pros and cons.

  5. Describe the SCAN policy for disk scheduling and its pros and cons. Show solutions for the mentioned drawbacks.

7 - File management¶

  1. What is a file? What are its main attributes?

  2. What is a file system? What are its main tasks?

  3. Describe sequential, direct and index access file methods.

  4. Descrivere le caratteristiche salienti del file system UNIX.

  5. What is an in-memory file system. Describe some example.

  6. Describe the main approaches for file allocation, discussing their pros and cons.

  7. Describe hard and soft links in a file system.

  8. Describe the chained allocation for files.

7 - File management¶

  1. Si consideri un file system con allocazione a catena ed accesso indicizzato. Dato un file:

  • contente 1M record;

  • il relativo indice ha taglia pari a 512 record e contiene 128 chiavi uniformemente distribuite nel file;

  • memorizzato su un dispositivo di massa, la cui taglia di un blocco è pari a 4096 record.

Calcolare la latenza massima e minima considerando che:

  • il tempo medio di accesso ad un blocco è 10ms;

  • l’accesso ad una chiave dell’indice in ram è pari a 1ms.

Show/Hide Solution

Assunzioni:

  • L’indice è memorizzato su disco

  • L’indice deve essere prima caricato in ram per poter essere utilizzato

  • RecordID e RiferimentoBlocco hanno pari taglia

Svolgimento:

  • Taglia di una chiave nell’indice = TagliaIndice/NumeroChiavi = 512R/128 = 4R

  • Taglia di un riferimento = 2R

  • Record utilizzabili all’interno di un blocco per memorizzare dati di file = Taglia di un blocco - Taglia di un riferimento = 4096-2 = 4094

  • Numero di blocchi per memorizzare il file = Numero di record nel file / record utilizzabili = ceil(10^6/4094) = 245

  • Distanza media tra due blocchi indicizzati = ceil(Numero di blocchi / numero di chiavi) = 2

  • Latenza minima: accesso ad un blocco indicizzato, indice già presente in RAM.

    • Risultato = Tempo di accesso a indice + Accesso al blocco indicizzato = 1ms +10ms = 11ms

  • Latenza massima: accesso al blocco più lontano da quello indicizzato, indice NON presente in RAM.

    • Risultato = Tempo di caricamento indice + Tempo di accesso a indice + (Numero di blocchi per indice)*(Tempo di accesso al blocco indicizzato) = +10ms + 1ms +2*10ms = 31ms

8- Programming¶

  1. Nei sistemi operativi UNIX, /dev/urandom è un dispositivo a caratteri (char device) virtuale in grado di generare numeri casuali. Nello specifico, l’operazione di lettura dal relativo file produce byte casuali. Scrivere un programma C che genera un file con contenuto interamente randomico. Il programma:

    • prende come parametri da linea di comando: un numero N e una stringa S da usare come nome del file da creare;

    • crea un file S contenente N byte randomici;

    • utilizza il dispositivo /dev/random come sorgente di numeri pseudo-casuali.

  2. Dato un file binario contenente un sequenza di 2^15 interi di tipo short, scrivere un programma che crea N processi o threads, i quali leggono il contenuto del file ed individuano il valore minimo e massimo contenuto nel file. Nel fornire una soluzione rispettare i seguenti vincoli:

  • ciascun intero non può essere letto da più di un thread/processo;

  • ciascun thread/processo può leggere il medesimo intero al più una volta;

  • ciascun thread/processo può allocare memoria nell’heap per al più 512 byte;

  • N è un parametro definito a tempo di compilazione o tramite linea di comando;

  • N è minore o uguale a 8;

  • è ammesso allocare di variabili globali (data) e locali (stack) per memorizzare tipi primitivi (puntatori, int, short, char, long, etc.) per al più 128 byte.

Per generare il file è possibile utilizzare la soluzione dell’esercizio 1.

  1. Scrivere un programma C invert che dato un file A ne inverte il contenuto e lo memorizza in nuovo file B. Il programma deve:

  • riportare il contenuto di A in memoria;

  • invertire la posizione di ciascun byte utilizzando un numero N di thread/processi concorrenti;

  • scrivere il risultato in un nuovo file B.

A, B e N sono parametri che il programma deve acquisire da linea di comando.

Solution: Github

  1. Si scriva il codice di una funzione C con la seguente interfaccia void tunnel(int descriptors[], int count) tale che, se eseguita, porti l’applicazione a gestire, per ogni file-descriptor dell’array descriptors l’inoltro del flusso dei dati in ingresso verso il canale di standard-output dell’applicazione. Il parametro count indica di quanti elementi è costituito l’array descriptors. L’inoltro dovrà essere attuato in modo concorrente per i diversi canali.

  2. Si scriva una funzione C con la seguente interfaccia void file_check(char *file_name, int num_threads). Tale funzione dovrà lanciare num_thread nuovi threads, in modo che ciascuno di essi legga stringhe dallo standard input, e per ogni stringa letta verifichi l’occorrenza di tale stringa all’interno di ciascuna riga del file il cui path è identificato tramite il parametro file_name, e stampi la stringa su standard output in caso affermativo.

  3. Scrivere un programma C in cui dato un file A, una stringa B e un intero N, vengano creati N thread/processi che cerchino se all’interno del file A esiste una linea uguale a B.
